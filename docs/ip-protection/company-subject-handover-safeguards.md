# 公司主体接入前权益保护清单

本文档用于在公司接入 ICP 备案、微信支付、客服、开票或推广主体前，明确合作边界。目标是支持公司合规运营，同时避免运营主体变化被解释为软件、源码、品牌或知识产权转让。

## 核心目标

1. 公司可以负责备案、支付、客服、推广和开票。
2. 项目早期软件、源代码、产品设计、原创文案、报告模板、品牌和相关证据链不因公司运营而当然归公司所有。
3. 项目赚钱后，个人仍能基于权属、授权或贡献获得收益。
4. 合作终止时，个人仍能继续持有和使用既有软件成果、源代码、品牌材料和证据链。

## 主体变更前最低确认

在备案或支付主体切换给公司前，至少保留一份书面确认。可以是正式备忘录、合同，也可以先通过邮件、飞书、微信文字确认，但正式合同更稳。

最低文字版本：

> 本项目早期版本、源代码、产品设计、原创文案、报告模板、品牌资料和相关技术文档由【个人姓名】主导形成。公司作为项目运营主体，负责 ICP 备案、微信支付、客服、推广和开票。公司基于项目运营需要使用上述软件和材料，但该等运营使用不构成软件著作权、商标权、作品著作权、源代码所有权或其他知识产权的转让。项目产生稳定收入、融资、转让、独占合作或引入新股东前，双方应另行书面约定授权范围、收益分配、费用承担、数据归属和退出安排。

## 不建议让步的点

- 不把软著权利人直接写成公司。
- 不把商标申请人直接写成公司，除非另有回购、授权费或股权安排。
- 不把“公司运营主体”写成“公司拥有全部知识产权”。
- 不把域名所有权、代码仓库 owner、服务器 root、支付密钥、软著账号全部交给公司单方控制。
- 不把个人证据包、身份证明、软著草稿、私有说明发给公司。

## 可以让步的点

- 公司作为 ICP 主办单位。
- 公司作为微信支付商户和结算主体。
- 公司作为客服、推广、开票和用户协议中的运营主体。
- 项目未产生稳定收入前，暂不收固定授权费。
- 公司可在运营范围内使用软件，但不得转让、独占或再授权给第三方，除非另行书面确认。

## 收益保护条款方向

正式协议里建议至少包含：

- 授权性质：非独占、不可转让、不可再授权。
- 授权范围：仅限本项目、指定域名、指定业务、指定公司主体。
- 收益安排：项目稳定收入后，另行确认授权费、技术服务费、利润分成或股权权益。
- 融资/出售：融资、转让、并购、独占合作前，必须先处理知识产权归属和授权收益。
- 终止机制：合作终止、公司停运、重大违约或股权控制变化时，个人可收回授权或要求公司停止继续使用。
- 资料返还：终止后公司应配合导出用户依法可迁移的数据、运营记录和必要交接资料。
- 既有成果：终止后个人仍可继续持有、使用、修改、复制、部署和商业化既有软件成果。

## 退出和拿回机制

合作做不下去时，至少要能做到：

1. 个人继续拥有既有源码和知识产权证据。
2. 公司停止以“拥有者”口径对外宣传该软件。
3. 公司不得阻止个人继续开发、部署或申请软著/商标。
4. 公司配合处理备案、支付、客服、用户协议和隐私政策主体切换。
5. 双方按书面约定处理公司运营期间新增的内容、用户数据和收入结算。

## 和合作方沟通的温和说法

> 下周主体切换主要是为了备案、支付和开票合规，不是现在谈分钱。我希望先把材料口径统一：公司负责运营，我继续负责产品和技术。这样对公司也安全，后续融资、平台审核、合作和税务都不会因为权属不清被卡。等项目真正有稳定收入或融资机会，再把收益分配和授权协议补正式。
