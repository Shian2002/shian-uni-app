# 时安解忧屋知识产权保护包

本文档用于先把项目权属证据、软著材料、商标和备案/支付边界整理清楚。它不是法律意见，正式提交和签署前建议让知识产权律师或熟悉互联网业务的律师过一遍。

## 当前建议路线

1. 个人先固定软件权属证据：源码、提交记录、版本号、截图、产品说明、设计文档。
2. 个人申请软件著作权，权利人优先写自然人。
3. 对品牌名、Logo、关键产品名做商标查询和申请。
4. 首页文案、报告模板、Logo/UI 设计等内容做作品登记或电子存证。
5. 公司需要 ICP 备案、微信支付、推广和客服时，只确认公司是运营主体，不把软件所有权写给公司。
6. 项目未产生稳定收入前，先用温和版《项目权属与运营主体确认备忘录》，不要一开始谈授权费和退出。

## 本目录文件

- `software-copyright-materials.md`：软件著作权申请材料清单和本项目填写建议。
- `ownership-operation-memo-template.md`：适合早期合作阶段使用的温和版备忘录模板。
- `trademark-filing-payment-checklist.md`：商标、ICP备案、微信支付和对外表述检查表。
- `operation-subject-change-note.md`：备案、支付和运营主体变更时的工程边界说明。
- `provenance-ledger.md`：项目形成过程、源代码范围、证据包和主体变更边界台账。
- `company-subject-handover-safeguards.md`：公司接入备案、支付和运营主体前的权益保护清单。
- `control-and-exit-playbook.md`：仓库、域名、部署、证据快照和合作退出控制权预案。

## 证据包导出

在项目根目录执行：

```bash
bash scripts/create_ip_evidence_package.sh
```

脚本会在 `artifacts/ip-evidence/` 下生成一个压缩包和一份元数据文件。`artifacts/` 已在 `.gitignore` 中，不会默认提交到仓库。

导出后建议把压缩包同步保存到至少两个位置，例如个人网盘、移动硬盘或可信时间戳/电子存证平台。不要把 `.env`、数据库、密钥、生产用户数据放进证据包。

## 不建议现在做的事

- 不建议把软著权利人直接写成公司，除非公司股权、控制权、退出和授权收益已经谈清楚。
- 不建议在对外材料写“公司拥有全部知识产权”，可以写“公司负责运营”。
- 不建议把域名、代码仓库所有权、服务器最高权限全部交给公司账号。
- 不建议把微信支付商户号主体和软件所有权混为一谈。
