## 紫微斗数

后端基于 `iztro-py>=0.3`：

- `solar_date`: `1990-1-27`
- `lunar_date`: `一九九〇年正月初一`
- `chinese_date`: `己巳 丁丑 壬辰 乙巳`
- 十二宫数量：`12`
- 命主：`文曲`

`SylarLong/iztro@2.5.8` JS：

- `solarDate`: `1990-1-27`
- `lunarDate`: `一九九〇年正月初一`
- `chineseDate`: `庚午 戊寅 壬辰 乙巳`
- 生肖：`马`
- 五行局：`水二局`
- 命主：`文曲`
- 十二宫数量：`12`

`Renhuai123/ziwei-doushu`：

- 代码依赖 `iztro@^2.5.8` 和 `lunar-javascript`
- 其算法入口直接调用 `astro.bySolar(...)`
- 结论：预计基础干支会跟 `iztro` JS 一致。

判断：

- 紫微在年/月柱上存在明显流派/库差异：后端 `iztro-py` 与八字按节气月对齐；`iztro` JS 对本春节/立春边界样例按另一套紫微历法口径输出 `庚午 戊寅`。
- 这不应直接判成 bug，需要产品决定：紫微页是否要坚持“和八字四柱一致”，还是跟随主流 `iztro` JS 口径。如果要改，影响会很大，需要单独设计迁移和回归样例。

## 结论

明确一致：

- 八字四柱：后端、`lunar-python`、`sxtwl`、`china-testing/bazi` 一致。
- 黄历/农历互转：后端、`lunar-python`、`sxtwl` 一致。
- 六爻本卦/变卦/动爻/世应：后端与 `bopo/najia` 一致。
- 塔罗牌库数量：后端与 `ekelen/tarot-api` 一致。

已知外部源问题：

- 问真八字公开接口本样例时柱返回 `庚子`，后端已本地修正为 `乙巳`。问真接口可用但不能无条件信任时柱。

需要产品决策的差异：

- 紫微：`iztro-py` 与 `iztro` JS 对春节/立春边界样例年柱、月柱口径不同。
- 奇门：后端与 `qfdk/qimen` 排法/分界规则差异大，应先明确采用拆补法、置闰法、转盘/飞盘、节气分界规则，再谈修正。

建议下一步：

1. 增加“玄学规则说明”页面或接口字段，标注每个工具采用的规则来源。
2. 紫微增加 2 组非边界样例和 2 组春节/立春边界样例，决定是否要与 `iztro` JS 对齐。
3. 奇门增加排法选择说明：拆补法/置闰法、阳遁阴遁分界、转盘/飞盘。
4. 问真 API 只作为八字年/月/日柱、大运参考源；时柱继续用本地算法。
