# 八字专业盘参考样本对照流程

目标：用你合法取得并认可的参考排盘结果，批量验证时安本地八字算法是否与参考口径一致。

## 原则

- 不在代码中内置第三方接口地址、账号、密钥或返回内容。
- 不自动抓取、绕过、批量请求第三方服务。
- 参考结果只作为人工录入的测试样本，用来校验本地算法口径。
- 对照报告只记录差异，不直接证明谁对谁错；边界样例需要人工判断采用哪套规则。

## 取样建议

第一批至少准备 30 组：

- 普通日期：10 组。
- 子时样例：5 组，覆盖 23:00、23:30、00:00、00:30。
- 节气边界：8 组，重点是立春、惊蛰、清明、立夏前后 2 小时。
- 性别顺逆：4 组，阳年男、阳年女、阴年男、阴年女。
- 历史年份：3 组，覆盖 1960 年以前、1990 年附近、2020 年以后。

第二批再扩到 100 组，用来做上线前信心验证。

## 字段优先级

必须高度一致：

- 年柱、月柱、日柱、时柱。
- 大运干支序列。
- 起运岁数。

需要人工确认口径：

- 起运具体文字，例如“出生后几年几个月几天起运”。
- 胎元、命宫、身宫。
- 神煞、旺衰、格局、调候用神。

已知容易产生差异的位置：

- 立春当天是否精确到时刻。
- 月柱是否按节令精确到时刻。
- 23:00 夜子时是否换日。
- 是否使用真太阳时。
- 起运算法里的天、月、小时折算规则。

## 使用方法

生成模板：

```bash
python3 scripts/compare_bazi_reference.py --write-template artifacts/bazi-reference/template.csv
```

复制模板为样本表：

```bash
cp artifacts/bazi-reference/template.csv artifacts/bazi-reference/cases.csv
```

在 `cases.csv` 中填写：

- `birth_time`: `YYYYMMDDHHMM`，例如 `199001271030`。
- `gender`: `男` 或 `女`。
- `use_solar_time`: 如果参考盘没有使用真太阳时，填 `false`。
- `reference_*`: 填入你认可的参考排盘结果；不知道的字段留空，脚本会跳过。

运行对照：

```bash
python3 scripts/compare_bazi_reference.py artifacts/bazi-reference/cases.csv --report artifacts/bazi-reference/report.md
```

判断标准：

- 报告里“差异字段数”为 0，说明已填写的参考字段都和本地算法一致。
- 有差异时，先不要直接改算法；把样例归类到“普通样例”还是“边界样例”。
- 普通样例差异优先修算法。
- 边界样例差异先定产品口径，再改算法。
