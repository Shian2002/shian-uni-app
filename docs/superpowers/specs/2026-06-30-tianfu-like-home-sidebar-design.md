# Tianfu-like 首页侧栏设计

日期：2026-06-30

状态：待用户复核后执行

## 目标

把首页左侧栏改成更接近 Tianfu / Codex 的轻量导航：默认少打扰，打开后只承载会话与命主管理，不再把低频入口堆到侧栏里。

这版设计取代 `2026-06-30-collapsible-home-sidebar-design.md` 中关于侧栏内容结构的部分。可折叠行为继续保留，但内容结构按本文执行。

## 参考行为

实机观察 Tianfu 后采用以下规则：

- 宽屏默认不是完整文字侧栏，而是一条很窄的图标 rail。
- 宽屏点击侧栏按钮后，完整侧栏推开主内容，不加遮罩。
- 半屏或手机默认完全收起。
- 半屏或手机点击侧栏按钮后，左侧抽屉覆盖页面，右侧内容变暗或轻微模糊，不压缩主界面。
- 侧栏内容以“新建对话、命主列表、收藏对话、历史对话”为主，不使用“工作台”分组。

## 侧栏内容

展开后的完整侧栏：

```text
时安解忧屋        [收起]

新建对话

命主列表          [+]
  当前命主 / 选择命主

收藏对话
  暂无收藏 / 收藏的问题

历史对话
  最近一条问题
  再早一点的问题

底部：设置 / 通知 / 回首页
```

设计要求：

- 不再出现 `工作台` 标题。
- 不再出现 `时安agent` 菜单项；顶部导航可以继续用 `时安agent` 作为当前频道标识。
- `新建对话` 是侧栏第一主动作。
- `命主列表` 可以露出当前命主；没有命主时显示 `选择命主`。
- `收藏对话` 没有内容时显示 `暂无收藏`。
- `历史对话` 最多露出 2 条最近对话，避免侧栏过长。
- 底部仅保留三个轻入口：设置、通知、回首页。

## 移出侧栏

以下内容不再占用左侧栏主空间：

- 常用术数
- 八字、奇门、紫微、六爻、梅花、择吉等术数入口
- 积分中心大入口
- 社区
- 关于我们
- 专属日历
- 更多帮助
- 完整历史列表

这些入口统一由顶部 `更多` 菜单承接。积分可以继续在顶部状态区或 `更多` 菜单中出现，但不作为侧栏大入口。

## 响应式状态

### 宽屏默认

- 默认显示窄图标 rail。
- rail 中只保留可识别的导航图标，不显示大段文字。
- 主内容保持居中，不因为默认 rail 明显偏移。

### 宽屏展开

- 点击侧栏按钮后展开完整侧栏。
- 完整侧栏宽度控制在约 240-260px。
- 主内容整体向右适配，输入框和主要内容不被侧栏遮挡。
- 展开状态下不使用遮罩。

### 半屏和手机

- 默认完全收起。
- 顶部只保留 `时安agent` 和 `更多` 两个主要入口，登录/主题按现有规则处理。
- 点击侧栏按钮后展示左侧抽屉。
- 抽屉宽度不超过视口的 82%。
- 右侧内容使用半透明遮罩，点击遮罩关闭。
- 不压缩主内容，避免半屏时页面挤成不可用状态。

## 交互

- 点击 `新建对话`：开启新的问事会话，并关闭移动端抽屉。
- 点击 `命主列表`：进入命主/档案列表；点击子项可选中当前命主。
- 点击 `收藏对话`：进入收藏对话列表或展开收藏摘要；没有收藏时不跳空页，可以显示 `暂无收藏`。
- 点击 `历史对话`：进入历史对话列表或展开最近记录；侧栏内最多展示两条。
- 点击底部设置：打开现有设置面板。
- 点击底部通知：后续接通知中心；当前没有通知中心时可以保持禁用或轻提示。
- 点击回首页：返回营销首页。
- 点击收起按钮：宽屏回到窄 rail，半屏/手机关闭抽屉。

## 视觉

- 沿用现有时安浅色、米白、金棕色调，不引入新的主色。
- 侧栏保持安静，不使用营销大卡片。
- 图标优先于文字说明，文字只用于入口名称和少量状态。
- 收藏、历史子项用轻背景短行，不做大卡片堆叠。
- 空状态保持短句，例如 `暂无收藏`、`登录后同步历史`。

## 验收标准

- 宽屏加载后默认是窄 rail，不是完整文字侧栏。
- 宽屏点击侧栏按钮后，完整侧栏展开并推开主内容。
- 半屏和手机加载后侧栏默认完全收起。
- 半屏和手机点击侧栏按钮后，出现遮罩抽屉，主内容不被挤压。
- 左侧栏不存在 `工作台`、`时安agent`、`常用术数`、`积分中心` 大入口。
- 左侧栏只出现新建对话、命主列表、收藏对话、历史对话、设置、通知、回首页。
- 顶部 `更多` 能承接术数、积分、社区、关于、日历、帮助等入口。
- 通过 Computer Use 在 Safari 中验收宽屏、半屏、侧栏打开/关闭和内容存在性。

## 非目标

- 不重做主页面 hero、输入框、对话流程。
- 不新增完整通知中心。
- 不新增收藏对话后端能力；如果当前没有收藏数据，先展示空状态或入口。
- 不改变登录、支付、积分计算逻辑。
