# 首页工作台可折叠左侧栏设计规格

日期：2026-06-30  
范围：`时安解忧屋` H5 首页工作台的左侧栏、顶部导航压缩规则、设置入口和设置半屏面板。

## Product Design Brief

目标是在首页工作台建立一个独立的可折叠左侧栏，视觉和交互参考 Tianfu Agent、Codex 的侧栏按钮模型：默认收起，点击侧栏按钮展开，再点击收起。展开时侧栏占据真实宽度并推开主内容，不复用旧的历史抽屉或遮罩式侧边栏。

视觉来源是用户提供的 Tianfu 全屏左栏、Tianfu 设置半屏面板、Codex 式侧栏按钮，以及当前 `时安解忧屋` 本地页面。交互目标是完整可用：侧栏开合、顶部“更多”、设置面板、导航入口都需要真实响应。

## 已确认决策

- 默认状态：侧栏默认收起，不是一直展开。
- 展开方式：采用推开式布局。侧栏展开后占据左侧宽度，主工作区向右让位。
- 内容密度：采用 Tianfu 的精简工作台模型，不放“最近问事”和“新手路径”。
- 顶部导航：半屏或手机宽度不足时，只保留“时安agent”和“更多”；其余入口收进“更多”。
- 右侧控制：主题按钮和登录按钮继续显示，不收进“更多”。
- 设置入口：设置按钮放在展开后的侧边栏内，点击后打开 Tianfu 式底部半屏设置面板。
- 架构边界：首页工作台侧栏不复用旧的 `tarotSidebarGlobal` / 历史抽屉逻辑，因为概念不一致。

## 非目标

- 不新增新的业务页面或路由。
- 不把历史记录抽屉改造成首页工作台侧栏。
- 不在首页半屏状态保留旧的左侧历史面板。
- 不在精简侧栏里展示“最近问事”和“新手路径”。
- 不把设置做成独立全屏页。
- 不改变支付、积分、登录、档案等业务数据逻辑。

## 响应式行为

### 默认收起

页面首次进入时，首页工作台侧栏为收起状态。收起状态只保留一个明确的侧栏按钮，位置靠近左上区域，形态参考 Codex 的小侧栏图标按钮。这个按钮属于首页工作台导航，不触发旧历史抽屉。

### 点击展开

点击侧栏按钮后，左侧栏从左侧展开，宽度建议为 268px 到 288px。主工作区向右让位，底部输入框、首页主视觉和中部快捷卡片都按剩余空间重新排布。再次点击按钮时，侧栏收起，主工作区恢复。

### 半屏和手机

半屏和手机默认仍然收起。展开时仍遵循“推开”原则，不显示旧的半透明历史抽屉。极窄手机上可以优先保证侧栏本身完整可读；主工作区被压缩后不需要在展开状态完整展示，但收起后必须恢复正常首页工作区。

### 全屏

全屏也保留同一个开关模型。产品可以在后续版本引入“记住展开偏好”，但本次默认不做记忆，避免用户误以为它永远展开。

## 侧栏内容设计

侧栏采用单列工作台结构，白色或近白底，右侧一条轻边线，金色只用于当前项、主要按钮和轻量强调。整体应克制、安静，和当前首页的东方命理气质一致，但不要做成一整块单色米金主题。

从上到下：

1. 品牌区：圆形品牌图标、`时安解忧屋`，可保留小字 `AI 命理工作台`。
2. 主要操作：`新建问事`，使用图标加文字，作为最明显的行动入口。
3. 工作台导航：`时安agent` 当前高亮；`档案列表`；`积分中心`。
4. 常用术数：`八字`、`奇门`、`紫微`、`六爻`，用两列轻量按钮。
5. 底部辅助：`设置`、`关于我们`、`回到首页`。

不展示：

- 最近问事
- 对话历史列表
- 新手路径
- 登录后同步历史记录提示

这些内容可以留在原本对应页面或未来独立历史入口里，不进入首页工作台精简侧栏。

## 顶部导航设计

在空间足够时，顶部导航可以保持现有主要入口。宽度不足时，顶部导航压缩为：

- 左侧：侧栏开关按钮
- 中间主要入口：`时安agent`
- 次级入口：`更多`
- 右侧：主题按钮、登录按钮

`更多` 菜单承载被收起的入口，包括 `专属日历`、`档案列表`、`社区`、`关于我们`，也可以补充 `积分中心` 和常用术数入口。`更多` 是导航聚合，不替代左侧栏；左侧栏仍是工作台上下文。

## 设置面板设计

点击侧栏内的 `设置` 后，打开底部半屏设置面板。面板参考 Tianfu 截图：

- 背景变暗并轻微虚化。
- 面板从底部出现，占据下半屏到三分之二屏高度。
- 顶部有小拖拽横条。
- 顶部三段：`个人账户`、`通用设置`、`更多帮助`，默认高亮 `通用设置`。
- 设置项包括主题、语言、自动跟随、隐私模式、联系我们。
- 主题和语言使用右侧下拉样式；自动跟随和隐私模式使用开关。
- 联系我们展开后显示邮箱和微信客服。

面板关闭方式包括点击遮罩、关闭按钮或 Escape。设置面板不负责打开旧历史侧栏。

## 架构约束

首页工作台需要自己的侧栏状态，例如 `homeSidebarOpen`。这个状态由首页或首页布局容器持有，顶部侧栏按钮只在首页语境下切换这个状态。

旧的 `tarotSidebarGlobal` / 历史抽屉可以继续服务原来的历史入口，但不能被首页工作台侧栏复用。避免出现“点击侧栏按钮后打开对话历史半屏”的旧行为。

建议把首页侧栏拆成独立组件，例如 `HomeWorkbenchSidebar`，并把设置面板拆成 `HomeSettingsSheet`。这样后续可以分别测试开合、内容和设置面板。

## 可访问性和状态

- 侧栏按钮需要有明确的 `aria-label`：展开侧栏 / 收起侧栏。
- 当前导航项需要有可见高亮和语义状态。
- 键盘焦点不能被隐藏在收起侧栏里。
- 打开设置面板时，焦点进入面板；关闭后焦点回到设置入口。
- 所有文字在半屏和手机宽度下不能溢出按钮或导航行。

## 验收标准

- 半屏首次进入：侧栏收起；左上有侧栏按钮；主内容不被旧历史面板遮住。
- 点击侧栏按钮：精简侧栏展开并推开主内容；再次点击收起。
- 展开侧栏内容只包含品牌、新建问事、工作台入口、常用术数、设置、关于我们和回首页。
- 半屏或手机顶部导航只保留 `时安agent` 与 `更多`，主题按钮和登录按钮仍显示。
- `更多` 菜单能访问被收起的主要导航入口。
- 点击侧栏内 `设置` 打开底部半屏设置面板，视觉接近 Tianfu 参考图。
- 不出现旧的对话历史半屏侧栏。
- 全屏、半屏、手机三类视口都通过真实浏览器点击验收。
- 实现完成后需要运行 `npm run typecheck`、`npm run lint`、`npm run build:h5`，并用 Computer Use 在本机浏览器复测关键路径。

## 后续实现顺序

1. 新建首页专属侧栏状态与组件边界。
2. 接通顶部侧栏按钮到首页侧栏开合。
3. 实现推开式布局和响应式压缩规则。
4. 调整顶部 `更多` 菜单内容。
5. 将设置入口迁入侧栏，并确认设置半屏面板状态。
6. 运行构建检查，再用 Computer Use 验收全屏、半屏和手机宽度。
