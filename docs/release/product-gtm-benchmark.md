## 1. 对标产品当前模式

Tianfu Agent 不是单纯的聊天页，而是“命理工作台”：

- 首页首屏把产品定义为 `The Metaphysics Agent`，主口号是 `Chart. Reason. Insight.`。
- 主 CTA 是 `Start Chat`，右上角 `Log In` 只是账号入口。
- 未登录点击 `Start Chat` 会打开登录/注册弹窗，支持 Global/China 切换、Google、GitHub、邮箱密码登录和注册。
- 页面反复强调 `250+ custom chart tools`、多 agent 协作、可视化推理、知识库、渐进推理、长期记忆、时间线锚定。
- 登录态产品更像“命理工作台”：命主、收藏、历史对话、新手任务、可视化抽屉和报告承接一起构成长期资产。

它的核心体验是：先建立命主和命盘，再围绕一个命主持续发问；不是每次都从零开始。

## 2. 变现结构

当前观察到的付费方案：

| 方案 | 积分 | 价格 | 权益 |
| --- | ---: | ---: | --- |
| 免费版 | 0 | ¥0 | 每日登录领 300 积分、1 个命主、1 个同时推演任务 |
| 入门版 | 3,000 | ¥9.9 | 3 个命主、3 个同时推演任务 |
| 标准版 | 12,000 | ¥36 | 5 个命主、5 个同时推演任务 |
| 专业版 | 30,000 | ¥68 | 10 个命主、10 个同时推演任务 |

积分消耗展示为：

- 短期问题：约 300 积分
- 普通长期问题：约 800 积分
- 复杂长期问题：约 1500 积分

商业逻辑是“免费额度试用 + 任务/签到促活 + 积分包/Pro 权益升级 + 多命主/并发任务扩容”。

## 3. 对我们产品的直接启发

时安解忧屋已有几个优势：

- 已有综合解读、问事引导、多术数、命盘档案、积分、历史记录、登录体系。
- 已有 H5、uni-app 多端基础，以及 Android/iOS/Harmony/Electron 发行骨架。
- 首页的“玄学产品感”比普通工具站更强，适合做品牌入口。
- 本轮已把顶部按钮调整为：`时安agent` 保留为主产品入口，未登录辅助按钮显示 `登录/注册`，登录后显示 `进入应用`。

目前相对 Tianfu Agent 的短板：

- 工作台感还不够强：命盘、对话、报告、历史之间的长期关系还没有形成强闭环。
- 付费展示不够产品化：用户看不到“不同问题大概要消耗多少积分、为什么要升级、升级后多了什么能力”。
- 报告承接不足：AI 输出更多停留在对话中，还需要可保存、可分享、可复购的“正式报告”形态。
- 新手成长链路不足：对标站用“新手任务 4% / 完成得积分”降低用户冷启动成本。
- 多端上架材料还需要围绕“隐私、账号注销、付费边界、内容免责声明”统一沉淀。
- 首屏仍偏“术数体系介绍”，还可以更早给出事业、感情、决策、年运这类可直接点击的问题模板。

## 3.1 按钮与登录链路判断

本轮本地验收确认：

- `时安agent` 没有丢，桌面和移动顶部都可见。
- 未登录点击 `时安agent` 会打开旧登录弹窗，弹窗保留 `密码登录`、`验证码登录`、`Gitee 验证登录`。
- 登录后点击 `时安agent` 会进入 `#/?app=1` 的应用态。
- 顶部辅助按钮已从固定 `进入应用` 改为未登录 `登录/注册`、登录后 `进入应用`。

产品判断：

- 未登录先登录的保护逻辑可以保留，Tianfu 的 `Start Chat` 也是打开登录弹窗。
- 但产品入口和账号入口必须分层：`时安agent` / `开始解读` 是产品入口，`登录/注册` 是账号入口。
- 后续不要用一个“登录”按钮替代 `时安agent`，否则用户会以为 agent 入口丢失。

## 3.3 本地复用线上登录的产品边界

`qa:login:local-online` 已证明本地前端可以使用线上 `/api/login` 和 `/api/me` 取得真实登录态。这解决的是“本地壳能不能拿回线上登录逻辑”的问题。

但它不证明完整 Agent 付费流已经通过。`qa:agent:local-online-login` 后续卡在 `test1` 积分不足，说明下一步要准备积分充足的审核/测试账号，或在 staging 做隔离积分策略。不要把“登录通过”包装成“完整问事到报告通过”。

## 4. 建议优先级

### P0：上架前必须补齐

- 隐私政策、用户协议、付费说明、内容免责声明、账号注销入口。
- App Store / 国内安卓市场的权限说明与实际使用一致。
- iOS 支付边界明确：数字内容/积分/订阅是否走 IAP，不能混用外部支付。
- GitHub Releases 保存每个平台产物、hash、审核记录。
- 积分充足的审核账号和 staging 隔离积分策略，否则审核员/测试用户会卡在首个 Agent 付费请求前。
- 账号注销截图必须能看到 `注销账号与删除数据` 和 `输入：注销账号`，否则 App Store / Google Play 审核证据不足。

### P1：提升转化

- 做一个“积分/会员方案”页，至少包含免费、入门、标准、专业四档。
- 首页和应用内都展示“短期问题/长期问题/复杂问题”的预计消耗。
- 增加签到、首问奖励、新手任务，形成每日回访理由。
- 将“问事引导 -> 推荐术数 -> 开始解读 -> 报告保存”做成连续链路。

### P2：提升复购和留存

- 每次完整解读生成一份报告卡：标题、命主、问题、术数、结论摘要、可分享链接。
- 做“命主人生档案”：长期问题、历史关键事件、已生成报告、未来待看节点。
- 建立“年度/月份运势报告”“合盘报告”“事业专项报告”等 SKU。

## 5. 营销链路建议

第一阶段不建议泛泛投“AI 算命”。更适合从具体场景切入：

- 事业：最近三个月事业机会、跳槽选择、创业时机。
- 感情：关系走向、复合判断、合盘分析。
- 决策：某个项目是否推进、合作对象是否合适。
- 年运：2026 年个人年度报告。

推荐链路：

1. 短视频/小红书/公众号内容用具体问题开头：例如“跳槽前先看这三个命盘信号”。
2. 落地页直接给一个可问的问题模板，不先解释功能。
3. 用户输入问题后进入问事引导，补足出生信息和场景。
4. 免费给摘要，完整盘面/多术数合参/正式报告消耗积分。
5. 结果页引导保存报告、分享、继续追问或购买更高额度。

## 6. 变现策略优先级

### 优先测试 1：免费额度 + 积分包

- 模式：每日签到给少量积分，短期问题低消耗，复杂合参和正式报告高消耗。
- 用户价值：先试再付，降低第一次付费阻力。
- 风险：如果免费额度太多，会压低付费；如果消耗规则不透明，会造成积分焦虑。
- 验证：上线积分说明页，观察“首页首问 -> 完整解读 -> 充值/兑换”的转化率。
- 成功线：首问后 24 小时内二次追问率提升 20%，完整解读页充值点击率达到 5%。

### 优先测试 2：正式报告 SKU

- 模式：对话摘要免费，完整报告消耗积分或单次购买。
- 用户价值：报告比聊天更适合保存、分享、复看和售后解释。
- 风险：报告质量必须稳定，否则会伤害信任。
- 验证：先做 H5 报告保存和报告预览，不急着接复杂支付。
- 成功线：完整解读后保存报告比例达到 30%，报告页带来至少 15% 继续追问。

### 暂缓测试：订阅会员

- 原因：当前产品还没有形成足够强的“每周/每月持续价值”，过早订阅会增加审核、客服和退款压力。
- 前置条件：新手任务、年度/月度报告、长期命主档案和历史复盘稳定后，再测试月卡或年卡。

### 暂缓测试：咨询师/专业版

- 原因：专业用户需要更高稳定性、批量命主管理、导出、售后和内容可解释性。
- 前置条件：报告体系、命主档案、桌面端工作台和数据备份能力稳定后再做。

## 11. 当前最该优化的三件事

1. **测试账号与积分策略**：上架前准备一组可交付审核的账号，账号内至少能完成 3 次标准 Agent 解读、1 次复杂合参、1 次账号注销演示。staging 环境要隔离短信、邮件、支付，但允许测试积分充足。
2. **商店截图重拍**：使用修正后的 `store:screenshots` 重新生成素材，人工确认第 3 张无遮挡、第 6 张露出注销区；正式提交当前批次前再用 Android/桌面真机重拍，iOS/鸿蒙后续恢复时单独补拍。
3. **报告资产闭环**：当前产品已经能从首页进入 Agent 和积分页，但“完整解读 -> 保存报告 -> 分享/复看 -> 继续追问”还不够强。报告页是后续首购、复购、客服解释和商店截图的共同支点。
