# 商店截图素材 Storyboard

> 更新时间：2026-06-17  
> 目标：商店截图不能只展示排盘表格，要展示“问事 -> 登录/注册 -> Agent 工作台 -> 积分说明 -> 专业工具 -> 隐私注销”的完整产品链路。

## 1. 生成命令

本地前后端启动后执行：

```bash
STORE_SCREENSHOT_BASE_URL=http://127.0.0.1:5173 npm run store:screenshots
```

脚本输出到 `artifacts/store-screenshots/<timestamp>/`：

- `summary.json`：截图结果、设备 preset、失败项。
- `README.md`：每张截图的用途说明。
- `*.png`：手机、大屏手机、平板、桌面截图。

这些图片是“商店素材候选”，当前批次正式提交前仍需用真实 Android、macOS、Windows 包各截一轮真机图；iOS 和鸿蒙后续恢复 active 后再补对应真机图。

当前可用候选包：`artifacts/store-screenshots/2026-06-16T19-01-27-100Z/`，共 14 张，`summary.json` 结果为通过。该包复用线上登录/注册接口生成真实会话，再用项目内 H5 tab 路由渲染 Agent、积分和个人中心截图；Agent 图不被登录弹窗遮挡，账号注销图能看到 `注销账号与删除数据`、`输入：注销账号` 和确认按钮。

## 2. 截图叙事顺序

| 顺序 | 截图 | 证明什么 | 商店用途 |
| --- | --- | --- | --- |
| 1 | 首页主入口 | 用户首屏能看到 `时安agent` 和具体问事入口，不是只有登录按钮 | Android/iOS/Harmony 第 1 张主图 |
| 2 | 旧登录弹窗 | 保留 `密码登录`、`验证码登录`、`Gitee 验证登录`，复用线上同款登录逻辑 | 审核账号路径说明、登录问题排查 |
| 3 | 时安 agent 工作台 | 核心体验是问事、命盘、术数选择和持续对话 | 商店核心功能图 |
| 4 | 积分会员与消耗说明 | 用户能理解免费额度、短期/长期/复杂问题消耗和升级价值 | 付费说明、转化图 |
| 5 | 八字排盘工具 | 展示专业术数工具能力，证明不是空壳聊天 | 功能截图、关键词覆盖 |
| 6 | 个人中心与账号注销 | 用户能找到账号、隐私和注销入口 | App Store / Google Play 审核材料 |

第 6 张截图必须滚动到 `注销账号与删除数据` 区域，并展开到能看到 `输入：注销账号` 的确认提示；只拍到个人中心顶部不能作为审核证据。

| 平台 | 必须截图 | 额外要求 |

| iOS | deferred | 后续恢复时补 iPhone 首页、Agent、积分页、iPad 横竖屏、账号注销；未接 IAP 前不要出现违规外部数字内容充值入口 |
| 鸿蒙 | deferred | 后续恢复时补手机首页、平板 Agent、权限弹窗、隐私入口；权限说明与实际触发时机一致 |
| macOS | 首次打开、窗口缩放、登录弹窗、Agent、积分页 | 标注签名/notarization 状态 |
| Windows | 安装页、开始菜单、首次打开、登录弹窗、卸载页 | 标注 EXE/MSI hash 和安全提示说明 |

## 4. 产品判断

按 GTM、变现和价值主张的视角，截图要让用户立刻明白：

- 这是能问事业、感情、决策、年运的东方命理 AI，不只是排盘工具。
- `时安agent` 是产品入口，`登录/注册` 是账号入口，两者不能互相替代。
- 免费体验能看到价值，但复杂合参、正式报告、长期档案要用积分承接。
- 结果应有下一步：继续追问、保存报告、查看历史、升级积分。
- 账号注销、隐私政策和权限说明要可见，避免商店审核卡在合规问题。
