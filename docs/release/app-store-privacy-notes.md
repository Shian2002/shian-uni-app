# 隐私、付费与审核口径

本文件用于统一 App Store、Google Play、国内安卓市场、华为/鸿蒙审核材料。这里不保存账号密码、证书、密钥或后台截图。

## 1. 应用定位

- 应用名称：时安解忧屋。
- 核心能力：传统文化工具、问事引导、八字/奇门/紫微等排盘、AI 综合解读、历史记录和报告保存。
- 审核描述建议：以传统文化、个人记录、娱乐参考和生活决策辅助表述，不承诺确定性预测、医疗、法律、金融收益。

## 2. 数据收集说明

| 数据 | 用途 | 是否必须 | 审核说明 |
| --- | --- | --- | --- |
| 账号信息 | 注册、登录、找回账号、保存历史 | 是 | 审核账号单独提供，不写入 Git |
| 出生信息 | 生成命盘和解读 | 核心功能需要 | 明确用于传统文化排盘和个性化解读 |
| 问题文本 | 生成 AI 解读 | 核心功能需要 | 提醒用户不要输入敏感隐私或违法内容 |
| 设备信息 | 登录安全、问题排查 | 可选或最小化 | 不提前申请无关权限 |
| 支付记录 | 积分、订单和售后 | 付费用户需要 | iOS/Google Play 按平台支付政策执行 |

## 3. 权限触发原则

- 首次启动不提前申请通讯录、定位、相册、麦克风等敏感权限。
- 只有用户主动使用相关能力时才触发权限申请，并展示用途说明。
- 如果当前版本没有实际使用某项权限，发行配置中不得声明该权限。
- 审核包、测试包、正式包的权限说明必须一致。

## 4. 账号注销与数据删除

当前实现：

- 个人中心已提供 `账号注销 -> 注销账号与删除数据` 入口。
- 前端会要求用户输入 `注销账号`，有密码账号还需输入当前密码，并弹出二次确认。
- 后端接口为 `POST /api/account/delete`，复用现有线上登录态，注销后清理命盘、历史、对话、收藏、社区内容等个人数据，并匿名化账号身份字段。
- 订单、充值和必要审计记录按合规要求保留，不在 GitHub 保存测试账号密码或真实用户隐私。

上架前仍必须确认：

- staging 和真机能找到注销入口，并能看到注销影响说明。
- 注销后旧账号密码无法继续登录，`/api/me` 返回游客态。
- 隐私政策、用户协议、商店 Data safety / App Privacy 与实际数据清理范围一致。
- App Store/Google Play 审核备注中写清楚入口路径。

## 5. 付费与积分口径

- Android 国内渠道：按各商店支付政策处理，测试包不得触达真实支付到账。
- Google Play：如果在 App 内销售数字内容、积分包或 AI 次数，需评估 Google Play Billing。
- iOS：未接入 Apple IAP 前，默认隐藏第三方数字内容充值入口；若销售积分或付费解读，应按 Apple IAP 改造。
- App Store / Google Play 审核包使用 `VITE_RELEASE_CHANNEL=appstore` 或 `VITE_RELEASE_CHANNEL=google-play` 构建；积分页会隐藏支付宝二维码、充值套餐、AI 次数套餐、创建充值订单和付款截图识别入口，仅保留积分消耗说明、签到、账本、历史和账号功能。
- 所有端都要展示积分消耗规则、退款/售后说明和订单记录入口。

## 6. 内容免责声明

建议在用户协议、结果页或报告页展示一致口径：

- 解读内容基于传统文化模型和 AI 生成，仅供娱乐、参考和自我记录。
- 不替代医疗、法律、金融、心理咨询等专业意见。
- 用户应自行判断并承担现实决策责任。

## 7. 审核备注模板

```markdown
审核账号：
登录方式：
核心路径：启动 App -> 登录 -> 首页输入问题 -> 进入时安 agent -> 生成综合解读 -> 查看历史/报告。
账号注销入口：个人中心 -> 账号注销 -> 注销账号与删除数据。
隐私政策 URL：https://shianjieyouwu.com/#/pages/legal/index?type=privacy
用户协议 URL：https://shianjieyouwu.com/#/pages/legal/index?type=terms
付费说明：当前版本 [隐藏第三方充值 / 使用平台官方支付 / 仅展示免费额度]。
权限说明：本版本仅在用户主动使用相关功能时申请权限。
```

当前 URL 由 `configs/release/legal-urls.json` 统一维护，已切到正式 HTTPS 域名；正式提交 Apple App Store、Google Play 和国内安卓市场前，应完成 `LEGAL_URL_CHECK_ONLINE=1 npm run store:legal-urls -- --strict` 线上可访问验证、商店后台 URL 截图和 APP 备案/不适用说明。

## 8. 审核账号资产准备

上架前审核账号必须能完成完整 Agent 演示，不能只证明“可以登录”。当前最低要求：

- 标准首问：3 次。
- 复杂合参：1 次。
- 追问：1 次。
- 账号注销路径：能看到入口和说明，但不要把真实审核账号提前注销。

检查命令：

```bash
npm run qa:audit-account
```

staging/local 可以用以下命令预置审核账号资产：

```bash
APP_ENV=staging DATABASE_URL=sqlite:////path/to/staging.db npm run qa:audit-account:seed -- --username audit_user --points 120 --ai-single 5 --ai-combo 2 --apply
```

安全边界：

- `qa:audit-account:seed` 默认 dry-run，不传 `--apply` 不会写数据库。
- 脚本默认拒绝 `APP_ENV=production`、`FLASK_ENV=production` 和已知生产库路径。
- 不用该脚本直接修改线上生产 `test1`；线上审核账号加权益必须通过后台管理员或受控运维流程完成，并留下审核记录。
- 证书、密码、审核账号明文不写入 GitHub。
