# 产品内容板块验收矩阵

> 更新时间：2026-06-15  
> 目标：把产品/GTM 判断落到可截图、可测试、可复盘的验收项。每个板块都要说明它服务哪个产品目标、需要什么截图证据、什么算通过。

## 验收口径

- `时安agent` 是产品主入口，`登录/注册` 是账号入口；任何改版都不能用登录按钮替代 agent 入口。
- 首页、Agent、积分、历史、工具、个人中心不是孤立页面，必须组成“问事 -> 解读 -> 积分/报告 -> 复看/追问 -> 合规账号管理”的链路。
- 商店截图分两类：转化截图负责说服用户，审核截图负责证明登录、隐私、注销、付费边界。
- 自动 H5 截图只能证明候选画面，当前批次 Android、macOS、Windows 必须用真实安装包重拍；iOS、鸿蒙延期到后续批次再补。
- 本地复用线上登录 smoke 只证明账号链路，不能证明 Agent 消费、报告保存、支付或真实用户验收通过。
- `qa:agent:entry-online-login` 证明入口和按钮状态，`qa:agent:local-online-login` 才证明完整 stream/消费流。
- `qa:audit-account` 必须先证明审核账号资产足够，否则不能把该账号交给应用商店审核员跑完整 Agent。

## 截图验收清单

- 首页首屏：`时安agent`、问事输入、登录/进入应用状态正确。
- 登录弹窗：密码登录、验证码登录、Gitee 验证登录都可见。
- Agent 工作台：输入区、命盘选择、术数选择、积分或历史入口无遮挡。
- 积分会员页：短期问题、长期问题、复杂问题、会员方案可见。
- 工具页：至少八字、奇门、紫微三类工具各一张。
- 历史/报告：正式报告功能完成后，必须加入报告保存、复看、继续追问截图。
- 个人中心：隐私政策、客服入口、账号注销入口和注销确认说明可见。

## 当前产品结论

- 当前产品已有“首页 -> 登录 -> 时安 agent -> 积分说明 -> 工具 -> 账号注销”的候选截图链路。
- 当前最弱的是“完整解读 -> 保存报告 -> 分享/复看 -> 继续追问”的报告资产闭环。
- 当前上架最大阻塞仍不是产品定位，而是真实平台安装包、真实用户回收、积分充足审核账号和商店后台证据。
