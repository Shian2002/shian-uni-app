## 三、开发与部署命令

### 本地开发
```bash
cd xuan-cet-tai
npm install              # 安装前端依赖
npm run dev:h5           # 启动 H5 开发服务器（端口 5173）
```

### 构建部署
```bash
npm run build:h5         # 构建 H5（自动执行 3 个 patch 脚本 + uni build）
bash deploy-to-server.sh # 一键部署到线上服务器
```

### 后端本地运行（可选）
```bash
cd backend
pip install -r requirements.txt
python app.py            # 启动后端（端口 5199）
```

## 四、架构要点与踩坑记录

### 4.1 双层架构：uni-app 前端 + Flask 后端

- 前端是 uni-app (Vue 3) 编译的 SPA，部署到 `/var/www/xuan-cet/` 由 Nginx 提供服务
- 后端 Flask 运行在端口 5199，Nginx 反向代理 `/api/*` 到后端
- 前端通过 `uni.request()` 调用后端 API（如 `/api/me`, `/api/bazi/...` 等）

### 4.2 Custom TabBar（关键！）

项目使用了 **custom tabBar**（`pages.json` 中 `"custom": true`），共 11 个 Tab 页面。**每个 Tab 页面都有独立的 Vue 实例和独立的 TopNav 组件实例**。

这意味着：
- 模块级变量（如 `var avatarLoaded`）在所有实例间共享
- 但每个实例的 `ref()` 是独立的
- 如果用模块级变量控制"只加载一次"，会导致其他实例的 ref 不被填充

### 4.3 uni-app H5 编译后的 DOM 结构

uni-app 编译 `<input>` 后会生成自定义元素：
```html
<uni-input class="form-input">
  <div class="uni-input-wrapper">
    <input class="uni-input-input" type="number" ...>
  </div>
</uni-input>
```

- `.form-input` 的 CSS 只应用到外层 `<uni-input>`
- 内层 `.uni-input-wrapper` 默认高度 0px
- 内层 `.uni-input-input` 默认 `height: 100%` 继承了 0px
- **必须额外设置 `.uni-input-wrapper` 和 `.uni-input-input` 的高度**

### 4.4 TopNav.vue 的全局事件处理

TopNav.vue 中注册了 **4 个全局捕获阶段事件监听器**：
1. `document.addEventListener('click', ..., true)` — 关闭下拉菜单
2. `document.addEventListener('touchstart', ..., true)` — 关闭下拉菜单（手机端）
3. `document.addEventListener('click', ..., true)` — 处理头像下拉菜单项点击
4. `document.addEventListener('touchstart', ..., true)` — 处理头像下拉菜单项触摸

这些监听器在捕获阶段执行，会在 Vue 事件处理之前运行。

### 4.5 `_xc_applyFrost` 函数

TopNav.vue 中有一个 `_xc_applyFrost` 函数，它会将下拉菜单 DOM 元素从 Vue 组件内部移动到 `document.body`。这会导致 **Vue 的所有事件绑定失效**（`@click.stop`, `@touchstart.stop` 等）。

因此，头像下拉菜单项的点击跳转不使用 Vue 事件绑定，而是通过 `data-href` 和 `data-action` 属性，在全局捕获阶段事件监听器中直接处理。

### 4.6 构建后 Patch 脚本

`npm run build:h5` 会按顺序执行：
1. `patch-vue-slot.js` — 修复 uni-app 编译后的 Vue slot 问题
2. `patch-tab-dom.js` — 修复 tabBar DOM 结构
3. `uni build` — 正式构建
4. `patch-html-css-order.js` — 将 CSS 链接移到 JS 脚本之前（避免 FOUC）

**修改这些脚本时要小心，它们是构建流程的关键部分。**

### 4.7 用户数据存储格式

- `uni.setStorageSync('xc_token', token)` — JWT token
- `uni.setStorageSync('xc_user', username)` — **纯用户名字符串**，不是 JSON
- `uni.setStorageSync('xc_avatar', avatarUrl)` — 头像 URL 字符串

### 5.2 潜在问题
- **部署脚本会覆盖 static 目录**：`deploy-to-server.sh` 用 rsync 同步 `dist/build/h5/static/` 到服务器，不会删除服务器上多余的文件，但如果本地的 `static/uploads/` 缺少文件，新上传的头像不会自动同步
- **shian 头像文件**：已手动上传 logo.png 为 `/var/www/xuan-cet/static/uploads/avatar_5_1779956998.png`，但数据库中的路径是 `/static/uploads/avatar_5_1779956998.png`。如果用户重新上传头像，文件名会变
- **后端 app.py 是单文件巨石**：约 8000+ 行，包含所有路由、业务逻辑、AI 调用等
- **无自动化测试**：没有 CI/CD，全靠手动测试验证
