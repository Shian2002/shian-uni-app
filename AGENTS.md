# xuan-cet-tai Codex 项目规则

## 基本协作

- 所有输出使用简体中文。
- 小任务直接执行，少讲过程，结果说明要清楚。
- 涉及数据库、部署、生产环境、账号积分、支付相关任务时，必须先核验证据和计划，再执行修改。
- 面对 2 个以上互不依赖的任务时，自动并行拆分处理。

## 浏览和验活

- 网页浏览、页面测试、截图、线上验活统一使用 gstack `/browse`。
- 本机 App、本机安装包、桌面壳、真实设备或本地系统 UI 相关测试，除脚本验证外，必须用 Computer Use 直接操作本机界面复测关键路径，例如点击、输入、下拉菜单、窗口缩放、安装/启动/退出等。
- Computer Use 只作为本机 UI 验活证据；不能替代线上 gstack 验活。涉及登录、上传、删除、支付、账号权限、系统设置等高风险 UI 操作时，必须按 Computer Use 确认策略在执行前单独确认。
- 不使用 Chrome MCP 或 `mcp__claude-in-chrome__*` 工具。
- 本地构建成功不等于线上生效；涉及发布时必须补线上验活证据。

## 代码库整洁

- 保持代码库干净，不保留临时文件、测试截图、构建产物、死代码或无用目录。
- 普通清理默认可以处理缓存、日志、构建产物、临时脚本和测试输出。
- 数据库、密钥、上传目录、备份目录默认保护，不随普通清理删除。
- 提交或交接前检查 `git status`，明确区分已有业务改动和本次新增改动。

## 长任务交接

- 长任务、生产排查、部署、数据库核验完成后，必要时写 handoff 文档。
- handoff 应包含当前目标、已完成事项、验证命令、风险、未决问题和下一步，方便新对话继续。
